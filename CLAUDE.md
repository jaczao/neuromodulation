# Neuromodulation for Continual Learning ŌĆö MLP Prototype

## Project
Thesis prototype testing a neuromodulation mechanism in an MLP, on two fronts:
1. Continual learning: does it reduce catastrophic forgetting on Split MNIST (alone, and stacked with the best baseline)?
2. Standard learning: does it improve, preserve, or degrade plain MNIST accuracy vs. a vanilla MLP?

The 1.5-day sprint is complete. Current work follows `@SPEC-proto-pt2.md` ŌĆö four sequential iterations of neuromodulation variants on the prototype. Read that SPEC at the start of each iteration session. After all four iterations, the project scaffolds to the definitive repo (which is when `THESIS-PLAN.md` will be created) and moves to GRU/CNN/ViT.

## Neuromodulation design
- `neuromod.py` holds a `Modulator` base class and a registry of variants, targets, and drivers.
- Three orthogonal axes, all config-selected, never hardcoded:
  - `--neuromod-variant {feedforward, stateful}` ŌĆö modulator architecture
  - `--neuromod-target {activation, plasticity, weight_mask}` ŌĆö where modulation acts
  - `--neuromod-driver {none, surprise, uncertainty, activation_stats}` ŌĆö what feeds the modulator
- `--use-neuromod` toggles the whole mechanism; OFF must reproduce the vanilla baseline numerically.
- Must compose with CL methods, e.g. `--method er --use-neuromod --neuromod-target plasticity --neuromod-driver surprise`.
- Iterations explore this grid **one axis at a time** per `@SPEC-proto-pt2.md`. Never combine multiple new mechanisms in one experiment.

## Non-negotiable rules
1. Never tune on any test set ŌĆö not the CL test sequence, not the official MNIST test set. CL tuning uses the validation sequence (seed=7); standard tuning uses a held-out split of the MNIST training set.
2. Standard and CL get SEPARATE hyperparameter sweeps (different optima ŌĆö epochs-per-task is meaningless for standard; CL wants smaller LR).
3. Identical tuning budget per method/variant.
4. Neuromodulation is config-selected and a one-flag swap (`--use-neuromod`) ŌĆö never tangle neuromod logic inside `model.py`. The `ModulatedLinear` wrapper is the one exception, and it must behave exactly like `nn.Linear` when no mask is supplied. Variant, target, and driver are args, not hardcoded. Must compose with CL methods.
5. Report mean ┬▒ std over 3 seeds for any final number.
6. After any edit to `data.py`, re-run `pytest tests/test_data.py`.
7. No hardcoded hyperparameters in training code ŌĆö everything routes through `configs.py`.
8. **Iteration discipline:** one new mechanism per iteration. Never combine. See `@SPEC-proto-pt2.md`.

## Known gotchas
- Normalize MNIST with whole-dataset stats, not per-task ŌĆö per-task normalization silently leaks task identity.
- EWC Fisher must be computed *after* finishing a task, before starting the next.
- ER buffer: update *before* the gradient step, sample with replacement, reservoir sampling for buffer fill.
- Standard val split comes from the TRAIN set, never the test set.
- `--use-neuromod` OFF must match vanilla exactly ŌĆö if it doesn't, the modulator is leaking into the base path.
- **Plasticity target + Adam:** multiplying `param.grad` by `╬▒` before Adam computes its moments is NOT the same as scaling per-parameter LRs ŌĆö Adam's running first/second moments are computed *from the scaled gradient*. Test with plain SGD first to isolate the mechanism, or scale the post-Adam update instead of the gradient.
- **Plasticity target trains the modulator ONLY via lookahead:** the literal SPEC hook (multiply `param.grad` by ╬▒ between `backward()` and `step()`, forward untouched) cannot train the modulator. With the forward untouched, the same-step loss is independent of ╬▒; the next-step loss depends on ╬▒ through `W_new = W ŌłÆ lr┬Ę(╬▒ŌŖÖg)`, but `optimizer.step()` writes W in place under `no_grad`, severing the ╬▒ŌåÆW_new autograd edge ŌåÆ `╬▒.grad = None`. Train it with a lookahead/first-order meta-gradient (`W_fast = W.detach() ŌłÆ lr┬Ę(╬▒ŌŖÖg)`, `L_meta` on `functional_call(model, W_fast)`, backward), then commit `W ŌåÉ W_fast.detach()`. (Iteration 1, rejected: even with ╬▒ free over [0,1] per-neuron, no retention signal in a current-task meta-loss ŌåÆ no forgetting reduction.)
- **Weight-mask target:** the mask must be **per-synapse** (one scalar per weight). A per-neuron mask (one scalar per row of W) is mathematically equivalent to pre-activation gain modulation and is therefore not a distinct mechanism.
- **Weight-mask on one hidden layer does NOT fix class-IL forgetting (Iteration 2, rejected).** The mask is in the forward graph so it trains by ordinary backprop (no lookahead) and learns full-range, task-differentiated masks, yet forgetting stays total (= Naive). Masking only net.2 leaves the first layer and the shared output head (net.4) to be overwritten, and class-IL forgetting is dominated by output-logit competition (van de Ven & Tolias 2019), which a hidden-layer mask cannot touch. Modulator LR ├Ś50 and low-rank r=16 do not change the result. `ModulatedLinear` with no mask must stay `allclose` to `nn.Linear` (parity).
- **Drivers:** every driver signal (surprise, uncertainty, activation_stats) must be **detached** before reaching the modulator. There must be no gradient path from a driver input back into the main loss. Implemented as a lag-1 control signal (computed from step t's loss/logits/activations, drives step t+1's mask via a `requires_grad=False` buffer; surprise EMA persists across tasks).
- **Drivers do NOT help on class-IL Split MNIST (Iteration 3, all rejected).** Under matched conditions on the weight_mask target, surprise Ōēł uncertainty Ōēł activation_stats Ōēł none Ōēł Naive (~0.198), even though all three are live, varying signals (mean |driver| Ōēł 0.19 / 0.04 / 12). They carry *novelty/difficulty* info, not *retention/importance* info ("what to protect"), and the mask still only gates one hidden layer while the shared head overwrites.
- **Stateful (GRU) modulator does NOT help either (Iteration 4, rejected).** weight_mask + surprise driven by a GRU whose hidden state persists across tasks: 0.1979 Ōēł Naive. Hidden 32 vs 64 identical. Detach the GRU state each step (truncated BPTT length 1) via a `clone()` of the persisted buffer, else an in-place-update autograd error fires. Tracking training dynamics is not the same as protecting the shared head.
- **All four pt2 iterations reject at Ōēł Naive (clean checklists). This is the SPEC "failure across all four" stop condition: do NOT add ad-hoc mechanisms.** Unifying cause: every variant acts on a hidden layer, but class-IL forgetting is dominated by the shared output head's logit competition (van de Ven & Tolias). Positive control: ER (replay) = 0.90 vs every hidden-layer neuromod Ōēł 0.198. See `prototype/iteration-notes.md` "Summary across all four iterations". Next move is a supervisor framing discussion, not more iterations.
- **pt3 Iteration 5 diagnostic CONFIRMED the head is the bottleneck.** Use `output_masking` (`none`|`loss`|`taskil`). Naive: class-IL 0.198 (forget 0.798) -> masked-train-loss only 0.389 (0.536) -> full task-IL 0.929 (0.070). So masked-loss (lever B) recovers a large but partial chunk (eval-time 10-way competition caps it); full head removal recovers almost everything. In task-IL the hidden-layer neuromod mechanisms still do NOT beat Naive, and on top of ER they add nothing (ER task-IL 0.996, +neuromod 0.995). `MaskedCE` masks **per sample** by label->task-pair (needed so ER replay isn't broken). pt3 retries (Iter 6-10) must reach the head / logits.
- **pt3 Iteration 6 (logit calibration, `target=logit`) rejected.** Per-sample FiLM on the 10 logits adds nothing: alone (no retention) it is EXACTLY Naive (0.1979) because a current-task-trained calibration just favors current classes; with masked-loss it equals masked-loss-alone (0.365 vs 0.378, modulator delta ~0); logit+ER (0.896) is below ER. **The masked loss (lever B), not the modulator, is what helps.** New bar for Iter 7-8: a real mechanism must beat `naive+masked-loss` (~0.38) standalone, not just Naive.
- **pt3 Iteration 7 (importance-gated plasticity, `target=importance`) rejected.** Online omega=sum(grad^2) gates per-param LR `alpha=1/(1+lambda*omega)` via grad-hooks (online EWC/MAS as a hard gate; lambda=0 == vanilla, verified). Standalone = 0.1977 = Naive (weight protection cannot fix class-IL, same as EWC=0.2014); +masked-loss 0.3975 vs 0.3777 (within noise); +ER 0.9035 vs 0.9023 (+0.001). No win. Reproduces "regularization fails class-IL" because the bottleneck is the head, not weight drift.
- **pt3 Iteration 8 (task-inferred routing, `target=task_route`, simplified HAT/lever C) rejected.** Masked-loss main net + task-inference net g(x); route eval output to inferred task. Standalone (naive): g catastrophically forgets (routing acc decays to chance 0.20) -> 0.199 = Naive. +ER (shared buffer trains g too): routing acc ~0.89 but routed ER 0.884 < ER 0.902, because hard routing makes g's ~11% errors unrecoverable (worse than ER's soft 10-way classification). Full HAT would hit the same inference ceiling + misroute penalty. Class-IL eval-time competition is not fixable by hard routing.
- **pt3 Iterations 9-10 rejected; ALL pt3 iterations now complete and reject (stop condition).** Iter 9 (`logit`+`recency` driver): a per-class recency signal does not rescue logit calibration (standalone 0.1979 = Naive; +ER 0.8948 < ER) - a recency bias cannot recover drifted old-class features. Iter 10 (`consolidation`: surprise-detected boundaries + EWC anchors): standalone 0.1974 = Naive (EWC fails class-IL; detector over-fires ~20 vs 4 true); +ER 0.9205 = +0.018 over ER, the closest pt3 result but under the +2pt bar. **Conclusion across pt2+pt3: no neuromod variant beats Naive by >=5pts on class-IL Split MNIST nor complements ER by >=2pts; replay is the only lever, neuromod adds ~0 on top. See `prototype/iteration-notes.md` "pt3 SUMMARY".** Next step is the supervisor framing discussion (task-IL reframe, or neuromod as a complementary stability term), NOT more iterations.

If activation modulation is ever revisited, prefer softplus (positive, unbounded above) or affine FiLM `(1+m) ŌŖÖ h + ╬▓` to preserve dense activations. Plasticity and weight-mask targets are fine with sigmoid (they gate learning/multiplication, not activations directly).

Standard: hidden-only gating preserves (~0.980), but `two_hidden_output` degrades by 0.40pt (per-sample gain on the 10 logits is unstable). Class-IL: no gate beats Naive standalone (~0.197; gain trained on the current task just favors current classes, same as Iter 6 logit), and on ER only `last_hidden` is neutral (+0.002) while more gating hurts ER, output gating worst (-0.029, the per-sample logit gain fights ER's replayed-head calibration). Uses 6-12x pt1 gain's params (314k-636k vs ~51k) yet does not beat pt1 gain: capacity was never the bottleneck. Gate output gain reaches the head but gain != memory. `ModulatedMLP.forward` now also calls `modulate_logits` (no-op for the pt1 GainModulator, used by direct_gain output gating).
