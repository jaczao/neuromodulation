# Neuromodulation for Continual Learning — MLP Prototype

## Project
Thesis prototype testing a neuromodulation mechanism in an MLP, on two fronts:
1. Continual learning: does it reduce catastrophic forgetting on Split MNIST (alone, and stacked with the best baseline)?
2. Standard learning: does it improve, preserve, or degrade plain MNIST accuracy vs. a vanilla MLP?

The 1.5-day sprint is complete. Current work follows `@SPEC-proto-pt2.md` — four sequential iterations of neuromodulation variants on the prototype. Read that SPEC at the start of each iteration session. After all four iterations, the project scaffolds to the definitive repo (which is when `THESIS-PLAN.md` will be created) and moves to GRU/CNN/ViT.

## Neuromodulation design
- `neuromod.py` holds a `Modulator` base class and a registry of variants, targets, and drivers.
- Three orthogonal axes, all config-selected, never hardcoded:
  - `--neuromod-variant {feedforward, stateful}` — modulator architecture
  - `--neuromod-target {activation, plasticity, weight_mask}` — where modulation acts
  - `--neuromod-driver {none, surprise, uncertainty, activation_stats}` — what feeds the modulator
- `--use-neuromod` toggles the whole mechanism; OFF must reproduce the vanilla baseline numerically.
- Must compose with CL methods, e.g. `--method er --use-neuromod --neuromod-target plasticity --neuromod-driver surprise`.
- Iterations explore this grid **one axis at a time** per `@SPEC-proto-pt2.md`. Never combine multiple new mechanisms in one experiment.

- Standard: `uv run python prototype/train.py --standard --seed N [--use-neuromod ...] [--no-wandb]`
- `--no-wandb` logs to stdout only — use it for sanity runs so they don't need W&B auth (or set `WANDB_MODE=offline`).
- W&B sweep configs live in `sweeps/` (create when needed).

## Non-negotiable rules
1. Never tune on any test set — not the CL test sequence, not the official MNIST test set. CL tuning uses the validation sequence (seed=7); standard tuning uses a held-out split of the MNIST training set.
2. Standard and CL get SEPARATE hyperparameter sweeps (different optima — epochs-per-task is meaningless for standard; CL wants smaller LR).
3. Identical tuning budget per method/variant.
4. Neuromodulation is config-selected and a one-flag swap (`--use-neuromod`) — never tangle neuromod logic inside `model.py`. The `ModulatedLinear` wrapper is the one exception, and it must behave exactly like `nn.Linear` when no mask is supplied. Variant, target, and driver are args, not hardcoded. Must compose with CL methods.
5. Report mean ± std over 3 seeds for any final number.
6. After any edit to `data.py`, re-run `pytest tests/test_data.py`.
7. No hardcoded hyperparameters in training code — everything routes through `configs.py`.
8. **Iteration discipline:** one new mechanism per iteration. Never combine. See `@SPEC-proto-pt2.md`.

## Metrics
### Standard learning (full MNIST)
- Test accuracy (mean ± std over 3 seeds), tuned via the standard sweep
- Two rows: vanilla MLP, neuromod MLP — neuromod must not materially hurt standard accuracy

### Continual learning (Split MNIST)
- Average final accuracy across all tasks (primary)
- Forgetting: mean over tasks of (max acc during training − final acc)
- Includes a combined "neuromod + best baseline" row to test complementarity
- Per-task accuracy trajectory (for plots, if time allows)

## Known gotchas
- Normalize MNIST with whole-dataset stats, not per-task — per-task normalization silently leaks task identity.
- EWC Fisher must be computed *after* finishing a task, before starting the next.
- ER buffer: update *before* the gradient step, sample with replacement, reservoir sampling for buffer fill.
- Standard val split comes from the TRAIN set, never the test set.
- `--use-neuromod` OFF must match vanilla exactly — if it doesn't, the modulator is leaking into the base path.
- **Plasticity target + Adam:** multiplying `param.grad` by `α` before Adam computes its moments is NOT the same as scaling per-parameter LRs — Adam's running first/second moments are computed *from the scaled gradient*. Test with plain SGD first to isolate the mechanism, or scale the post-Adam update instead of the gradient.
- **Plasticity target trains the modulator ONLY via lookahead:** the literal SPEC hook (multiply `param.grad` by α between `backward()` and `step()`, forward untouched) cannot train the modulator. With the forward untouched, the same-step loss is independent of α; the next-step loss depends on α through `W_new = W − lr·(α⊙g)`, but `optimizer.step()` writes W in place under `no_grad`, severing the α→W_new autograd edge → `α.grad = None`. Train it with a lookahead/first-order meta-gradient (`W_fast = W.detach() − lr·(α⊙g)`, `L_meta` on `functional_call(model, W_fast)`, backward), then commit `W ← W_fast.detach()`. (Iteration 1, rejected: even with α free over [0,1] per-neuron, no retention signal in a current-task meta-loss → no forgetting reduction.)
- **Weight-mask target:** the mask must be **per-synapse** (one scalar per weight). A per-neuron mask (one scalar per row of W) is mathematically equivalent to pre-activation gain modulation and is therefore not a distinct mechanism.
- **Weight-mask on one hidden layer does NOT fix class-IL forgetting (Iteration 2, rejected).** The mask is in the forward graph so it trains by ordinary backprop (no lookahead) and learns full-range, task-differentiated masks, yet forgetting stays total (= Naive). Masking only net.2 leaves the first layer and the shared output head (net.4) to be overwritten, and class-IL forgetting is dominated by output-logit competition (van de Ven & Tolias 2019), which a hidden-layer mask cannot touch. Modulator LR ×50 and low-rank r=16 do not change the result. `ModulatedLinear` with no mask must stay `allclose` to `nn.Linear` (parity).
- **Drivers:** every driver signal (surprise, uncertainty, activation_stats) must be **detached** before reaching the modulator. There must be no gradient path from a driver input back into the main loss. Implemented as a lag-1 control signal (computed from step t's loss/logits/activations, drives step t+1's mask via a `requires_grad=False` buffer; surprise EMA persists across tasks).
- **Drivers do NOT help on class-IL Split MNIST (Iteration 3, all rejected).** Under matched conditions on the weight_mask target, surprise ≈ uncertainty ≈ activation_stats ≈ none ≈ Naive (~0.198), even though all three are live, varying signals (mean |driver| ≈ 0.19 / 0.04 / 12). They carry *novelty/difficulty* info, not *retention/importance* info ("what to protect"), and the mask still only gates one hidden layer while the shared head overwrites.
- **Stateful (GRU) modulator does NOT help either (Iteration 4, rejected).** weight_mask + surprise driven by a GRU whose hidden state persists across tasks: 0.1979 ≈ Naive. Hidden 32 vs 64 identical. Detach the GRU state each step (truncated BPTT length 1) via a `clone()` of the persisted buffer, else an in-place-update autograd error fires. Tracking training dynamics is not the same as protecting the shared head.
- **All four pt2 iterations reject at ≈ Naive (clean checklists). This is the SPEC "failure across all four" stop condition: do NOT add ad-hoc mechanisms.** Unifying cause: every variant acts on a hidden layer, but class-IL forgetting is dominated by the shared output head's logit competition (van de Ven & Tolias). Positive control: ER (replay) = 0.90 vs every hidden-layer neuromod ≈ 0.198. See `prototype/iteration-notes.md` "Summary across all four iterations". Next move is a supervisor framing discussion, not more iterations.
- **pt3 Iteration 5 diagnostic CONFIRMED the head is the bottleneck.** Use `output_masking` (`none`|`loss`|`taskil`). Naive: class-IL 0.198 (forget 0.798) -> masked-train-loss only 0.389 (0.536) -> full task-IL 0.929 (0.070). So masked-loss (lever B) recovers a large but partial chunk (eval-time 10-way competition caps it); full head removal recovers almost everything. In task-IL the hidden-layer neuromod mechanisms still do NOT beat Naive, and on top of ER they add nothing (ER task-IL 0.996, +neuromod 0.995). `MaskedCE` masks **per sample** by label->task-pair (needed so ER replay isn't broken). pt3 retries (Iter 6-10) must reach the head / logits.
- **pt3 Iteration 6 (logit calibration, `target=logit`) rejected.** Per-sample FiLM on the 10 logits adds nothing: alone (no retention) it is EXACTLY Naive (0.1979) because a current-task-trained calibration just favors current classes; with masked-loss it equals masked-loss-alone (0.365 vs 0.378, modulator delta ~0); logit+ER (0.896) is below ER. **The masked loss (lever B), not the modulator, is what helps.** New bar for Iter 7-8: a real mechanism must beat `naive+masked-loss` (~0.38) standalone, not just Naive.
- **pt3 Iteration 7 (importance-gated plasticity, `target=importance`) rejected.** Online omega=sum(grad^2) gates per-param LR `alpha=1/(1+lambda*omega)` via grad-hooks (online EWC/MAS as a hard gate; lambda=0 == vanilla, verified). Standalone = 0.1977 = Naive (weight protection cannot fix class-IL, same as EWC=0.2014); +masked-loss 0.3975 vs 0.3777 (within noise); +ER 0.9035 vs 0.9023 (+0.001). No win. Reproduces "regularization fails class-IL" because the bottleneck is the head, not weight drift.
- **pt3 Iteration 8 (task-inferred routing, `target=task_route`, simplified HAT/lever C) rejected.** Masked-loss main net + task-inference net g(x); route eval output to inferred task. Standalone (naive): g catastrophically forgets (routing acc decays to chance 0.20) -> 0.199 = Naive. +ER (shared buffer trains g too): routing acc ~0.89 but routed ER 0.884 < ER 0.902, because hard routing makes g's ~11% errors unrecoverable (worse than ER's soft 10-way classification). Full HAT would hit the same inference ceiling + misroute penalty. Class-IL eval-time competition is not fixable by hard routing.
- **pt3 Iterations 9-10 rejected; ALL pt3 iterations now complete and reject (stop condition).** Iter 9 (`logit`+`recency` driver): a per-class recency signal does not rescue logit calibration (standalone 0.1979 = Naive; +ER 0.8948 < ER) - a recency bias cannot recover drifted old-class features. Iter 10 (`consolidation`: surprise-detected boundaries + EWC anchors): standalone 0.1974 = Naive (EWC fails class-IL; detector over-fires ~20 vs 4 true); +ER 0.9205 = +0.018 over ER, the closest pt3 result but under the +2pt bar. **Conclusion across pt2+pt3: no neuromod variant beats Naive by >=5pts on class-IL Split MNIST nor complements ER by >=2pts; replay is the only lever, neuromod adds ~0 on top. See `prototype/iteration-notes.md` "pt3 SUMMARY".** Next step is the supervisor framing discussion (task-IL reframe, or neuromod as a complementary stability term), NOT more iterations.
- **Activation target output range:** sigmoid silently sparsifies activations. If activation modulation is ever revisited, prefer softplus (positive, unbounded above) or affine FiLM `(1+m) ⊙ h + β` to preserve dense activations. Plasticity and weight-mask targets are fine with sigmoid (they gate learning/multiplication, not activations directly).

- **Direct-gain modulator (`target=direct_gain`, user-requested) rejected.** pt1 gain without the bottleneck/projection (a direct `Linear(784->width)` gain head per gated layer; gate `{last_hidden, two_hidden, last_hidden_output, two_hidden_output}`). Standard: hidden-only gating preserves (~0.980), but `two_hidden_output` degrades by 0.40pt (per-sample gain on the 10 logits is unstable). Class-IL: no gate beats Naive standalone (~0.197; gain trained on the current task just favors current classes, same as Iter 6 logit), and on ER only `last_hidden` is neutral (+0.002) while more gating hurts ER, output gating worst (-0.029, the per-sample logit gain fights ER's replayed-head calibration). Uses 6-12x pt1 gain's params (314k-636k vs ~51k) yet does not beat pt1 gain: capacity was never the bottleneck. Gate output gain reaches the head but gain != memory. `ModulatedMLP.forward` now also calls `modulate_logits` (no-op for the pt1 GainModulator, used by direct_gain output gating).
- **pt4 (standard regime): 5 of 9 iteration mechanisms are N/A by construction in single-task learning.** drivers, stateful, task_route, logit+recency, and consolidation all depend on task boundaries or a task sequence that does not exist in standard MNIST, so they either reduce to a runnable mechanism (driver/stateful->weight_mask, recency->logit) or to vanilla (consolidation), or are undefined (task_route). Do NOT manufacture a degenerate number for them; the "why" is the result. Group R (activation, weight_mask, logit, plasticity, importance) all PRESERVE vanilla accuracy (R3 logit marginally improves). Plasticity in standard uses an SGD main net (Adam-moments caveat), so compare it to the SGD-vanilla reference, not the Adam vanilla.
- **CL `--optimizer` was silently dropped (FIXED).** `main()`'s CL branch never copied `args.optimizer` into the config (only the `--standard` branch did), so every CL run used Adam regardless of `--optimizer sgd`. Fixed (the CL branch now applies it). Symptom that caught it: naive-SGD and naive-Adam were byte-identical at every lr. If you compare optimizers in CL, confirm the flag actually took effect.
- **SGD vs Adam is a huge confound under masked loss on class-IL (re-confirms the plasticity/SGD caveat).** With `output_masking='loss'` (1 seed): naive+masked-loss = **0.39 with Adam vs 0.63 with SGD** (forgetting 0.54 vs 0.12) — Adam's per-parameter scaling overwrites old-task weights far faster. Because the plasticity loop's main net is SGD, plasticity+masked-loss (0.628) looked like a big win but equals naive-**SGD**+masked-loss (0.630); the gating adds nothing. ALWAYS compare a neuromod mechanism to its **same-optimizer** naive+masked-loss baseline. Under matched optimizers none beats naive: Adam — gain 0.398≈naive 0.389, weight_mask 0.360 (hurts); SGD — plasticity 0.628≈naive 0.630, gain 0.575 (hurts), weight_mask 0.629≈naive. Masked loss (lever B) + SGD is the lever, not neuromod.
- **pt5 dispatch: `_is_pt5` must own the pt5 path; guard the legacy `_is_*` helpers that collide (FIXED).** pt5's `--neuromod-target plasticity` was silently intercepted by the legacy `_is_plasticity` branch (checked earlier in `cl_train`'s elif chain), so it ran the Adam-lookahead sigmoid modulator (tell: `[plasticity debug] alpha range [0.857, 0.982]` instead of `[pt5 debug]` and a binary {0,1} gate). - **pt5 Iteration 1 (disjoint task-id subnetworks): gain+ER is the first neuromod cell to beat replay (+10pts), accept-for-confirm; the rest reject.** SGD lr=1e-3 ep=5, 1 seed. Baselines naive-SGD+masked-loss 0.6296, ER-SGD 0.7226. **gain+ER 0.8264 (+0.104 vs ER, forget 0.0089)**: a disjoint gain gate freezes each task's private subnet (a gated-off unit gets zero grad on BOTH its incoming weights AND its outgoing head column, so the subnet + its head columns are preserved), the oracle selects it at eval (task 0 holds 0.997 across all 5 tasks), and ER calibrates the shared 10-way head that gain cannot touch. Complementary levers. gain standalone (masked-loss) 0.6225 ≈ naive+masked-loss: hidden retention is perfect but the newest task collapses to 0.001 (masked loss isolates each task's logits, so class-IL eval loses the newest classes; replay, not masked loss, calibrates the head). plasticity (-0.21 / -0.27) and weight_mask (-0.19 / **-0.61 catastrophic**) reject: the hard {0,1} freeze at 1/5 capacity is too aggressive, and weight_mask+ER explicitly masks net.4 per-task so replayed old-class samples pass through the wrong head synapses and scramble the logits (gain's implicit head-column freeze cooperates with replay; weight_mask's explicit head mask fights it). Oracle caveat: task-IL-style result on the class-IL metric (privileged task id at eval selects the subnet). 3-seed confirm of gain+ER deferred per SPEC.
- **pt5 granularity/scope axes (added, orthogonal sub-flags — NOT new targets).** `--neuromod-granularity {neuron,synapse}` (default `neuron`) applies to the `activation` and `plasticity` pt5 targets; `weight_mask` is always synapse and ignores it. `--neuromod-plasticity-scope {both,in,out}` (default `both` = legacy in+out coupling) is per-NEURON plasticity only. Three new cells: (1) per-neuron plasticity in-only/out-only via `param_factors(scope=)` (net.2.weight is the shared matrix: `in`→rows a1, `out`→cols a0, `both`→a1·a0); (2) per-synapse plasticity (`SynapsePlasticityDriverModulator.weight_grad_masks`) gates WEIGHT grads only per synapse, forward at full W, biases left plastic, layer set = `--neuromod-mask-layers`; (3) per-synapse gain (`activation`+`synapse`) = `TaskWeightMaskMLP(gain_form=<config>)`, the forward `(Gamma⊙W)x`. **Under a fixed binary P (iter 1 disjoint, iter 2 shared), per-synapse gain is numerically identical to `weight_mask`** (both reduce to `{0,1}⊙W`; `gain_gamma(raw, fixed=True)=raw`); they diverge only under the learned projection (iter 3, gain uses `1+raw`/`sigmoid`). Defaults reproduce the prior pt5 behaviour exactly (granularity=neuron, scope=both). Per-synapse variants raise if `--neuromod-mask-layers` is empty.
- **pt5 per-neuron mechanisms are layer-selectable (added).** Per-neuron plasticity: `--neuromod-plasticity-layers` (net.<idx> weight matrices, subset of {0,2,4}; 0=in→h0, 2=h0→h1, 4=h1→out) + ONE global `--neuromod-plasticity-scope {in,out,both}` per run. Only hidden neurons carry α, so a (layer,side) with no α is a no-op: net.0 has no `out` (input has no α), net.4 has no `in` (output has no α). So a single global scope CANNOT mix sides across layers (e.g. `{0:in, 2:out, 4:out}` is not expressible in one run; user chose this simpler encoding over a per-layer side spec). `param_factors(alphas, scope, layers)` handles it; default `(0,2,4)`+`both` == legacy coupling. Per-neuron gain: `--neuromod-gain-layers` (activations 0=h0, 2=h1, 4=output logits; default `0,2`=both hidden); `GainDriverModulator` gained a `gate_layers` arg + `modulate_logits` (its per-layer buffers are now `P_h0`/`P_h1`/`P_out`, renamed from `P_0`/`P_1`). Gating logits (4) under a disjoint fixed P was ASSUMED to == task-IL output masking, but that is FALSE (see below): `build_disjoint_proj` assigns the 10 class-columns to tasks RANDOMLY, not aligned to the true class→task map, so the output gate zeros the WRONG classes. Per-synapse gain/weight_mask stay on `--neuromod-mask-layers`.
- **pt5 output-logit gating (`--neuromod-gain-layers …,4`) is BROKEN under the current random projection — collapses to chance (verified).** `P_out = build_disjoint_proj(5, 10, seed+2)` is a RANDOM balanced partition of the 10 output columns across tasks, NOT the true Split-MNIST class→task map (task t = classes (2t,2t+1)). So the eval gate for task t keeps 2 random class-columns, almost never t's real classes: e.g. seed 0 → task 0 keeps {3,6} (true {0,1}) → correct logits zeroed → 0%. Result: gain(h0,h1,out) ≈ 0.10 (naive) / 0.19 (ER), pattern `[0.000, 0.500, 0.000, 0.000, 0.000]` (only the one task whose gate accidentally overlaps a true class scores ~0.5). To make logit-gating actually equal task-IL masking, P_out must be BUILT from the sequence (columns of task t = its class ids), not a random disjoint projection. Until then, do NOT gate layer 4. Hidden-only gain(h0,h1) is the working variant.
- **pt5 loop now honors `--optimizer` (was hardcoded SGD "always, Methodology 6").** Default is still SGD *for the pt5 code path*, BUT `CLConfig.optimizer` defaults to `"adam"`, so a pt5 run that omits `--optimizer` now uses ADAM — always pass `--optimizer sgd` to reproduce prior Methodology-6 pt5 numbers. Forgetting →~0 with gain everywhere (0.001–0.008) vs baselines 0.13–0.55; the freeze always works, only the head-competition WIN differs, and SGD in 5 epochs leaves logits marginal (huge variance). Adam standalone gain (0.9949, no replay) even beats the ER baseline (0.893) — the disjoint-subnet freeze is a separate lever from replay. Reproduction: seed 42 reproduces iter-1 bit-exact (naive+gain 0.6225, er+gain 0.8264), clearing the recent gain-layer/bias/optimizer edits of any regression. All ORACLE (task-id at eval) → task-IL-style on the class-IL metric. **Buffer matters for the ER cells**: iter-1 used er_buffer_size=1000; the CLI default is 200, which alone drops er+gain-SGD from 0.826 to ~0.65 — always pass `--er-buffer-size 1000` to compare with iter-1.
- **pt5 optional per-neuron BIAS modulation (`--neuromod-modulate-bias`, added, orthogonal toggle — default OFF = parity).** Applies to the per-SYNAPSE mechanisms that previously left biases fully plastic: per-synapse gain/`weight_mask` (`TaskWeightMaskMLP`) and per-synapse plasticity (`SynapsePlasticityDriverModulator`). ON → each listed layer also gets an INDEPENDENT per-neuron projection `P_bias_l: (T, d_out)` in its own seed namespace (`build_fixed_proj(..., seed + BIAS_PROJ_SEED_OFFSET=10_000 + idx)`, never aliased to the weight P). weight_mask/gain gate the bias in the FORWARD `(Γ⊙W)x + (β⊙b)` (β uses the same `gain_form` as W; `ModulatedLinear.forward` gained a `bias_mask` arg, `bias_mask=None` == parity); per-synapse plasticity adds `net.<l>.bias` grad gates to `weight_grad_masks()`. Per-NEURON gain already freezes biases implicitly (γ=0 → zero activation → zero bias grad) and per-neuron plasticity already gates biases on the `in` side (`param_factors`), so the toggle is a no-op there and is not wired to them. **Honest caveat baked into the docstrings:** biases are a STRUCTURALLY shared resource (every task's forward adds ALL biases, unlike per-synapse weights), so a disjoint `P_bias` only protects the ~1/T of biases each task owns AND (mask gate) suppresses ~(T-1)/T of biases in the forward — aggressive; the toggle exists to ablate it, not because it's known to help. The output head bias should stay plastic under ER (freezing it fights replay's head recalibration, cf. weight_mask+ER −0.61); this toggle can reach it via `--neuromod-mask-layers 4`, so don't enable bias modulation on the head in +ER runs unless ablating that. Default OFF reproduces prior pt5 numbers exactly.
